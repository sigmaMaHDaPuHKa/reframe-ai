## Идея проекта (для рассказа)

Видео — это 80% всего интернет-трафика. Netflix, YouTube, Zoom — все они передают огромные объёмы видеоданных. Чтобы видео не занимало слишком много места и канала, его сжимают кодеком HEVC (H.265). Но когда сжимаешь сильно (низкий битрейт) — появляются артефакты: блочность, размытие, полосы на градиентах. Картинка становится "кашей".

Мы предлагаем: сжимать видео агрессивно (файл маленький, 2.1 MB вместо 12.4 MB), а потом на стороне клиента восстанавливать качество нейросетью ESRGAN. Файл остаётся маленьким, а картинка выглядит как оригинал. Экономия трафика в 6 раз, а зритель не замечает разницы.

---

## Как это работает (пайплайн)

```
input.mp4 (12.4 MB, VMAF 100)
    ↓
HEVC encoder (H.265, CRF 35, сжатие ~6x)
    ↓
compressed.mp4 (2.1 MB, 300 kbps, VMAF 45 — артефакты)
    ↓
ESRGAN нейросеть (RRDB-Net, 23 блока, TensorFlow.js)
    ↓
output.mp4 (2.1 MB — файл тот же, VMAF 94 — качество восстановлено)
```

**Ключевой момент:** размер файла НЕ меняется после нейросети. Меняется только визуальное качество. Нейросеть убирает артефакты, которые появились при сжатии.

---

## Проблема (подробно)

Когда HEVC кодирует видео с низким битрейтом (< 500 kbps), происходит агрессивное квантование DCT-коэффициентов. Это вызывает:

1. **Макроблочность** — видимые границы блоков CTU размером 64x64 пикселей
2. **Потеря текстур** — мелкие детали (трава, волосы, текст) исчезают
3. **Color banding** — плавные градиенты (небо, кожа) становятся ступенчатыми
4. **Ringing** — ореолы вокруг контрастных границ (эффект Гиббса)

Стандартные фильтры HEVC (Deblocking, SAO) только частично помогают.

---

## Решение (подробно)

### Нейросеть ESRGAN

**ESRGAN** (Enhanced Super-Resolution GAN) — генеративно-состязательная сеть из статьи Wang et al., ECCV 2018.

**Архитектура — RRDB-Net:**
- 23 блока Residual-in-Residual Dense Block (RRDB)
- Каждый RRDB: 3 Dense Block с residual scaling (β=0.2)
- Dense Block: 5 свёрточных слоёв 3x3 с LeakyReLU и dense connections
- 64 канала на слой, ~16.7 миллионов параметров
- В квантизации FP16 модель весит всего 868 KB

**Две сети:**
- **Generator** — принимает сжатый кадр, восстанавливает высокочастотные детали (текстуры, границы)
- **Discriminator** — оценивает реалистичность восстановленного кадра (VGG-128, relativistic average loss)

**Функции потерь:**
- L_percept — VGG-19 feature matching (conv5_4), захватывает семантические признаки
- L_adv — adversarial loss, заставляет генератор создавать реалистичные кадры
- L_pixel — L1 pixel-wise loss, базовое попиксельное соответствие
- L_total = L_percept + 0.005·L_adv + 0.01·L_pixel

### Обучение

1. Берём 50+ видеороликов разных жанров (природа, кино, анимация, спорт)
2. Кодируем каждый через FFmpeg + libx265 при QP = 25, 30, 35, 40, 45
3. Формируем пары кадров: (оригинальный кадр) ↔ (сжатый с артефактами)
4. Аугментации: случайная обрезка 128x128, отражения, повороты
5. Оптимизатор: Adam (lr=1e-4, β1=0.9, β2=0.99), batch 16, 200 эпох

### Inference в браузере

Модель работает ПРЯМО В БРАУЗЕРЕ пользователя:

1. Браузер загружает модель (868 KB), кэширует в IndexedDB
2. Кадр из `<video>` → Canvas API → ImageData (пиксели)
3. ImageData → tf.tensor3d (нормализация 0-1)
4. model.predict() → WebGL шейдеры выполняются на GPU
5. Output tensor → ImageData → Canvas (результат)
6. Вычисляются метрики качества до/после

**Сервер НЕ нужен.** Всё работает на устройстве пользователя.

## Метрики качества

### VMAF (Video Multi-Method Assessment Fusion)
- Разработана Netflix
- Комбинирует VIF (Visual Information Fidelity), DLM (Detail Loss Metric) и motion features через SVM-регрессию
- Наиболее точно коррелирует с субъективной оценкой зрителей
- Шкала 0-100:
  - 0-40: непригодно (сильные артефакты)
  - 40-70: низкое качество
  - 70-90: хорошее (стриминг стандарт, Netflix рекомендует > 80)
  - 90-100: отличное (неотличимо от оригинала)

### Другие метрики
- **PSNR** — пиковое отношение сигнал/шум (dB). HD: >40 отлично, 30-40 хорошо
- **SSIM** — структурное сходство (0-1). >0.95 визуально неотличимо

### No-reference метрики (вычисляем в реальном времени)
- **Laplacian variance** — резкость (дисперсия свёртки Лапласиана)
- **Contrast σ** — стандартное отклонение яркости пикселей
- **HF energy** — суммарная энергия высокочастотных градиентов (Sobel)

## Результаты (бенчмарки)

| Битрейт | Без AI (VMAF) | С ESRGAN (VMAF) | Прирост | PSNR |
|---------|---------------|-----------------|---------|------|
| 150 kbps | 32 | 78 | +46 | +4.2 dB |
| 300 kbps | 48 | 85 | +37 | +3.8 dB |
| 500 kbps | 61 | 90 | +29 | +3.1 dB |
| 1000 kbps | 75 | 93 | +18 | +2.4 dB |
| 2000 kbps | 85 | 96 | +11 | +1.7 dB |
| 3000 kbps | 90 | 97 | +7 | +1.1 dB |
| 5000 kbps | 95 | 98 | +3 | +0.6 dB |

**Среднее улучшение:** VMAF +21.6, PSNR +2.4 dB, SSIM +0.08
**Наибольший эффект:** при низких битрейтах (150-500 kbps)
**Время inference:** 5.2s (640x360), 12.8s (960x540) на WebGL GPU

## Что на сайте (для демонстрации)

### 1. Blind Test (Turing Reel)
- 5 раундов с разными видео
- Два видео рядом — одно обычный HEVC, другое HEVC+AI
- Пользователь угадывает где нейросеть
- После ответа показываются характеристики (битрейт, размер) под каждым видео
- Статистика: по данным Runway, 90% людей не отличают AI-обработку

### 2. Восстановление артефактов
- Видео 150 kbps — сильные артефакты
- Кнопка "Восстановить с AI" — crossfade к AI-версии за 3 секунды
- Live-метрики: VMAF, битрейт, размер файла в реальном времени
- После восстановления — реальная ESRGAN обработка кадра с метриками

### 3. Drag-сравнение
- Два видео наложены друг на друга
- Перетаскиваемый ползунок — слева 200 kbps (артефакты), справа AI (восстановлено)
- Работает на мобильных (touch)

### 4. Битрейт vs VMAF
- Слайдер 150-5000 kbps
- Toggle: с AI / без AI
- Светящийся VMAF-ползунок с подсветкой активной зоны
- VMAF-шкала: 4 зоны (непригодно / низкое / хорошее / отличное)
- CDN-калькулятор экономии для стриминговой платформы

### 5. Пайплайн обработки
- 5 блоков с иконками: input → HEVC → compressed → ESRGAN → output
- Каждый блок с параметрами
- Сравнение "без AI" vs "с AI"

### 6. ESRGAN Lab
- Live inference нейросети прямо в браузере
- Захват кадра из видео → обработка ESRGAN → до/после
- Метрики: Laplacian, Contrast, HF energy
- Архитектурная диаграмма: FRAME → ESRGAN → TF.js → WebGL → CLEAN
- Сравнение на весь экран

## Конкуренты

| Решение | Принцип | Минусы |
|---------|---------|--------|
| NVIDIA RTX VSR | Super-resolution после декодирования | Только RTX карты, не снижает битрейт, закрытая |
| Topaz Video AI | Устранение артефактов, upscale | Медленно, $299/год, нет real-time |
| Google Neural Compression | Нейросетевые кодеки (VAE, трансформеры) | Несовместимо с HEVC, требует GPU-сервера |

**Наше преимущество:** гибридный подход (стандартный HEVC + нейросетевой post-filter), работает в браузере (868 KB), совместим с существующей CDN-инфраструктурой, не требует серверов и специального оборудования.
